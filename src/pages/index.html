RESPONSUM API example

An example of how to integrate with the RESPONSUM API to accompany or online documentation

(optional) Configure webhook

If you want to see the webhook in action, you can configure a new webhook under settings > integrations > webhooks with the public url to this page /webhook
For any POST request to that endpoint, the body will be logged.
So if you configure that webhook to listen to information-management-system subsystem and the Item created and Item base data updated actions, you'll see the webhook message come in as you run the below operations.

GET /uuids/{amount}

Generate new UUIDS according to the Responsum format

GET /country filtered on name

Get a country model
Using param-filter we can pass in a JSON stringified object of which each property must match exactly with the model we're filtering for
In the next operation we'll use this when creating a new information-management-system

POST /create

Create one or more models and their links
In this operation we'll create a new information-management-system with a link to an existing country

PATCH /update

Update one or more models and their links
In this operation we'll lookup the IMS we created in the previous operation and then update the description and the country link
